# ВСТАНОІЛЕННЯ MySQL DEBIAN 10
  Оновіть списки пакетів:
	➜  ~ sudo apt update

  Встановіть пакет MariaDB Server:
	➜  ~ sudo apt install mariadb-server

  Під час встановлення пакету MariaDB Server вас можуть
  запросити ввести пароль для користувача root бази даних.

  Після встановлення MariaDB Server потрібно запустити службу
  за допомогою наступної команди:
 	➜  ~ sudo systemctl start mariadb

  Щоб перевірити, чи служба MariaDB Server успішно запущена, введіть команду:
	➜  ~ sudo systemctl status mariadb

  Для налаштування бази даних MariaDB використовуйте наступну команду:
	➜  ~ sudo mysql_secure_installation

  Ця команда запустить майстер налаштування безпеки, який допоможе налаштувати
  різні параметри, такі як пароль користувача root, відключення анонімного входу,
  видалення тестових баз даних і т.д.


# КОНФІГУРУВАННЯ MySQL DEBIAN 10
  MariaDB має конфігураційний файл з назвою my.cnf, який зазвичай знаходиться в директорії /etc/mysql/. Цей файл містить
  ряд налаштувань, які визначають параметри роботи сервера баз даних, такі як порт, каталог баз даних, обсяг пам'яті,
  яку можна використовувати для буферизації запитів і т.д.

Щоб внести зміни в конфігураційний файл MariaDB на Debian 10, слід діяти наступним чином:

1. Відкрийте файл конфігурації my.cnf за допомогою текстового редактора:
	➜  ~ sudo nano /etc/mysql/mariadb.conf.d

2. Внесіть необхідні зміни в конфігураційний файл. Наприклад, якщо ви хочете змінити порт, на якому працює MariaDB,
   з 3306 на 3307, знайдіть рядок зі словом "port" і змініть його значення на 3307.

3. Збережіть зміни і закрийте файл.

4. Перезапустіть службу MariaDB за допомогою команди:
	➜  ~ sudo systemctl restart mariadb

# СТВОРЕННЯ БАЗИ ДАНИХ

1. Зайдіть в інтерактивний режим командної оболонки MySQL за допомогою команди:
	➜  ~ sudo mysql

2. Створіть нового користувача бази даних і базу даних за допомогою наступних команд:
	➜  ~ CREATE DATABASE dbname;
	➜  ~ CREATE USER 'username'@'localhost' IDENTIFIED BY 'password';
	➜  ~ GRANT ALL PRIVILEGES ON dbname.* TO 'username'@'localhost';
	➜  ~ FLUSH PRIVILEGES;
   Замість "dbname", "username" та "password" введіть ім'я бази даних, ім'я користувача та пароль відповідно.

3. Вийдіть з інтерактивного режиму командної оболонки MySQL, введіть "exit".

4. Щоб забезпечити те, що MariaDB буде запускатися при перезавантаженні системи, виконайте наступну команду:
	➜  ~ sudo systemctl enable mariadb

5. Щоб перезапустити MariaDB, виконайте наступну команду:
	➜  ~ sudo systemctl restart mariadb
